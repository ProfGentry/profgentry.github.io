1 Understanding VMware & Virtualization

üß† Understanding VMware

Before we click buttons, we need to understand what‚Äôs actually happening.

üí° What is Virtualization?

Virtualization is the creation of a simulated (virtual) computer environment that runs inside your physical computer. It allows you to run multiple different operating systems at the same time without them interfering with your main system.

üß± Why Cybersecurity Uses Virtual Machines

Security professionals rely on VMs to:

  • Safely run offensive tools and exploits
  • Isolate malware samples
  • Test configurations and defenses
  • Recreate networks and targets for practice

‚öôÔ∏èüß© How VMware Works

VMware simulates core hardware components such as:

  • CPU
  • RAM
  • Storage
  • Network adapters

This lets your virtual machine behave just like a real computer, but inside a window.

üóÇÔ∏è Types of Virtual Machines

  • üñ•Ô∏è Full OS machines (e.g., Kali Linux)
  • üì¶ Specialized appliances (pre-built security distros)
  • üß™ Lightweight testing environments

üõ°Ô∏è Why We Use Kali Linux for TryHackMe

Kali includes tools such as Nmap, Burp Suite, Hydra, Gobuster, and Metasploit, making it an ideal penetration testing environment for platforms like TryHackMe.

4 Understanding VM Networking Modes

‚è±Ô∏è ~10 minutes

In this part, you will understand how your virtual machine communicates with the outside world and with your host system. These modes determine whether your Kali VM can access the internet, talk to your local network, or stay isolated for safe testing.

üåê NAT (Recommended)

NAT (Network Address Translation) allows your VM to share your host computer‚Äôs internet connection. The VM gets a private internal IP address, and VMware acts like a router translating traffic.

Why this matters:

  • Safest option for beginners.
  • Ideal for TryHackMe VPN access.
  • Keeps your VM hidden from your real LAN, reducing risk.

Analogy: NAT is like your VM using your host‚Äôs Wi-Fi without ever knowing the actual password.

üåâ Bridged

Bridged mode places your VM directly on your physical network as if it were its own device with its own IP address.

Why this matters:

  • Useful when simulating real networks.
  • Helpful for labs where the VM must act as a separate machine.
  • Less isolated, so slightly higher risk if misused.

Analogy: The VM becomes a ‚Äúreal roommate‚Äù on your network‚Äîvisible and discoverable.

üöß Host-Only

Host-Only mode isolates your VM from the internet entirely. The VM can only communicate with your host computer.

Why this matters:

  • Great for malware analysis when you want nothing leaking out.
  • Useful for offline testing and isolated labs.
  • Not suitable for TryHackMe because it requires internet access.

Analogy: Host-Only is like airplane mode for your VM, except it can still talk to your host.

üîë Key Takeaways

Each mode controls your VM‚Äôs ‚Äúreach‚Äù:

  • üåê NAT ‚Üí Internet access, safe, hidden behind your host.
  • üåâ Bridged ‚Üí Internet + LAN visibility, appears as a separate device.
  • üöß Host-Only ‚Üí No internet, only host-to-VM communication.

Understanding these choices ensures your VM behaves correctly for different cybersecurity scenarios.

5 Connecting Kali to TryHackMe

‚è±Ô∏è ~15‚Äì20 minutes

In this part, you will connect your Kali VM to TryHackMe using a VPN configuration file. This allows your VM to securely join the TryHackMe network so you can reach lab machines from inside Kali.

Step 1 üõ†Ô∏è: Confirm Internet Access from Kali

Make sure your VM network is working before adding the VPN.

  1. Start your Kali VM in VMware.
  2. In VMware, confirm the VM‚Äôs Network Adapter is set to NAT.
  3. Inside Kali, open a terminal.
  4. Run: 
    ping -c 4 8.8.8.8
  5. You should see replies from 8.8.8.8. Press Ctrl+C to stop.
  6. Optional: Open Firefox and browse to https://tryhackme.com.

Step 2 üõ†Ô∏è: Update Kali‚Äôs Package List

Refresh Kali‚Äôs knowledge of available software.

  1. In the Kali terminal, run: 
    sudo apt update
  2. Enter your password if prompted.
  3. Wait for the command to complete.

Step 3 üõ†Ô∏è: Install OpenVPN

Install the VPN client that will create the tunnel to TryHackMe.

  1. In the same terminal, run: 
    sudo apt install openvpn -y
  2. Wait while Kali downloads and installs OpenVPN.
  3. When the prompt returns with no errors, OpenVPN is installed.

Step 4 üõ†Ô∏è: Download Your TryHackMe VPN Configuration File

Get a unique .ovpn file that links your VM to your TryHackMe account.

  1. In Kali, open Firefox.
  2. Go to https://tryhackme.com and log in.
  3. Go to the Access / VPN section.
  4. Choose the OpenVPN option.
  5. Download your .ovpn configuration file.
  6. Verify it appears in your Downloads folder in Kali.

Step 5 üõ†Ô∏è: Navigate to the VPN File in the Terminal

Move into the folder where your .ovpn file is stored.

  1. In the Kali terminal, run: 
    cd ~/Downloads
  2. List the files to verify the .ovpn file is present: 
    ls
  3. Look for a filename ending in .ovpn (e.g., thm-region-yourusername.ovpn).

Step 6 üõ†Ô∏è: Connect to TryHackMe Using OpenVPN

Create a secure VPN tunnel from your VM to TryHackMe.

  1. Still in the Downloads folder, run (replace the filename with your actual VPN file): 
    sudo openvpn yourfile.ovpn
  2. Enter your password if prompted.
  3. Watch the terminal output as OpenVPN connects.
  4. When you see Initialization Sequence Completed, the VPN connection is active.
  5. Do not close this terminal window while using TryHackMe; closing it will terminate the VPN.

Step 7 üõ†Ô∏è: Verify the VPN Connection

Confirm that your VM is successfully on the TryHackMe network.

  1. Open a new terminal window in Kali (leave the OpenVPN window running).
  2. Run: 
    ip a
  3. Look for an interface named tun0 ‚Äì that‚Äôs your VPN tunnel.
  4. In your browser, start a TryHackMe machine and note its IP address.
  5. From the new terminal, try: 
    ping -c 4 <THM_machine_IP>
  6. If you receive replies, your Kali VM is successfully connected to the TryHackMe network.

Step 8 üõ†Ô∏è: Disconnect When You‚Äôre Done

Safely close the VPN tunnel.

  1. Return to the terminal window where OpenVPN is running.
  2. Press Ctrl+C to stop the OpenVPN process.
  3. You should see messages indicating the connection is terminating.
  4. The tun0 interface will disappear, and your VM will go back to normal NAT internet use.
