1 Understanding Hyper-V & Virtualization

‚è±Ô∏è ~10 minutes

üß† Understanding Hyper-V

Before building a virtual machine, it is important to understand what the hypervisor does and how it manages resources on the host system.

üí° What is Virtualization?

Virtualization is the process of creating a simulated computer environment (a virtual machine) that runs on top of physical hardware. A single physical device can host multiple virtual machines, each with its own operating system and software, running independently from the host and from one another.

üß± Why Cybersecurity Uses Virtual Machines

Security professionals rely on virtual machines to:

  • Safely run offensive security tools and exploit code
  • Isolate and analyze potentially malicious software
  • Simulate networks and systems for defensive testing
  • Recreate lab environments that can be reset as needed

‚öôÔ∏èüß© How Hyper-V Works

Hyper-V is Microsoft‚Äôs type-1 hypervisor built into Windows. It manages and abstracts hardware such as:

  • Virtual CPUs (vCPUs)
  • Virtual memory
  • Virtual hard disks (VHD/VHDX files)
  • Virtual network adapters and switches

From the guest operating system‚Äôs perspective, the virtual machine behaves like a physical computer, even though it is sharing the underlying hardware with other guests.

üóÇÔ∏è Types of Virtual Machines

  • üñ•Ô∏è Full OS machines (e.g., Parrot Security OS)
  • üì¶ Pre-built security appliances (firewalls, IDS/IPS virtual images)
  • üß™ Lightweight, disposable test systems for experiments

üõ°Ô∏è Why We Use Parrot for Hack The Box

Parrot Security OS is a Debian-based distribution designed for penetration testing, digital forensics, and privacy. It includes a curated set of tools for enumeration, exploitation, web testing, and post-exploitation. This makes it well-suited for interacting with Hack The Box machines in a controlled lab environment.

2 Enabling Hyper-V on Windows

‚è±Ô∏è ~15‚Äì20 minutes

üß© What You Are Doing

In this section you will verify that your system supports virtualization and enable the Hyper-V feature on Windows. If Hyper-V is already installed and working, you may proceed directly to the Parrot VM creation section.

üìù Step-by-Step Instructions

1. Verify Hardware Virtualization Support

  • On your Windows host, press Ctrl + Shift + Esc to open Task Manager.
  • Select the Performance tab and click CPU.
  • Confirm that Virtualization is listed as Enabled.
  • If it is not enabled, you will need to enable virtualization in BIOS/UEFI before proceeding.

2. Enable Hyper-V Using Windows Features

Use the built-in Windows feature dialog to enable Hyper-V.

  1. Open the Start menu and search for Turn Windows features on or off.
  2. In the dialog, locate and check the boxes for:
    • Hyper-V
    • Ensure both Hyper-V Management Tools and Hyper-V Platform are selected.
  3. Click OK. Windows will install the required components.
  4. Restart the system when prompted.

3. Launch Hyper-V Manager

After the restart, confirm that Hyper-V is available.

  1. Open the Start menu and search for Hyper-V Manager.
  2. Launch Hyper-V Manager.
  3. Verify that your local computer appears in the left-hand pane under Hyper-V Manager.

Note: On some editions of Windows, Hyper-V is not available. In that case, consult your instructor for an alternate hypervisor.

3 Setting Up a Parrot Security VM

‚è±Ô∏è ~35‚Äì45 minutes

In this part, you will create a new virtual machine that runs Parrot Security OS. Follow each step carefully and pay attention to the choices you make, as they affect performance, storage, and network behavior.

3.1 üì• Download the Parrot ISO

First, obtain the installer image (ISO) for Parrot Security OS.

  1. On your host (Windows) system, open a web browser.
  2. Navigate to the official Parrot download page: https://www.parrotsec.org/download/
  3. Select a Security edition ISO (64-bit), preferably the stable release.
  4. Download the ISO file and save it in a location that is easy to remember (for example, Downloads or C:\ISOs\Parrot).
3.2 üèóÔ∏è Create a New Virtual Machine in Hyper-V

You are now creating the ‚Äúcomputer within a computer.‚Äù

  1. Open Hyper-V Manager.
  2. In the right-hand Actions pane, click New > Virtual Machine‚Ä¶.
  3. Review the introductory text and click Next.
3.3 üñ±Ô∏è Specify Name and Generation

Provide a meaningful name and select the virtual machine generation.

  1. Enter a name such as Parrot for HackTheBox.
  2. Optionally change the storage location (e.g., C:\VMs\Parrot-HTB).
  3. Click Next.
  4. Select Generation 2 if supported by the ISO and your host; otherwise, select Generation 1. When in doubt, follow instructor guidance.
  5. Click Next.
3.4 üß† Assign Memory

Decide how much RAM the virtual machine should use.

  1. Assign at least 4096 MB (4 GB) of startup memory if your host has ‚â• 8 GB.
  2. You may enable Use Dynamic Memory for this virtual machine so Hyper-V can adjust usage as needed.
  3. Click Next.
3.5 üåê Configure Networking (Initial Selection)

You will attach the VM to a virtual switch for network connectivity.

  1. If a Default Switch or appropriate External virtual switch exists, select it here.
  2. If no switch is available, choose Not Connected and you will configure networking later in Part 4.
  3. Click Next.
3.6 üíæ Create a Virtual Hard Disk

Create storage for the Parrot VM.

  1. Keep the option Create a virtual hard disk selected.
  2. Specify a name and location for the VHDX file, or accept the defaults.
  3. Set the size to at least 40 GB to accommodate tools and updates.
  4. Click Next.
3.7 üî•‚û°Ô∏èü¶ú Attach the Parrot ISO

Tell Hyper-V to boot from the Parrot installer ISO.

  1. On the Installation Options page, choose Install an operating system from a bootable image file.
  2. Click Browse‚Ä¶ and select the Parrot ISO you downloaded earlier.
  3. Click Next, then review the summary.
  4. Click Finish to create the VM.
3.8 üõ†Ô∏è Start the VM and Run the Parrot Installer

Install Parrot onto the virtual hard disk.

  1. In Hyper-V Manager, right-click your new Parrot for HackTheBox VM and select Connect‚Ä¶.
  2. In the Virtual Machine Connection window, click Start.
  3. When the Parrot boot menu appears, select the appropriate Install or Graphical Install option.
  4. Follow the on-screen prompts:
    • Select language, region, and keyboard layout.
    • Set a hostname (for example, parrot).
    • Create a user account and password; record this securely.
    • Use guided partitioning to install to the entire virtual disk.
    • Allow the installer to install a bootloader when prompted.
  5. Wait while the installer copies files and configures the system.
3.9 üîÅ Reboot into the Parrot Desktop

Complete the installation and verify that the desktop environment works.

  1. When prompted, select Continue to reboot.
  2. If the VM still boots from the ISO, power off, remove the ISO from the virtual DVD drive in the VM‚Äôs settings, and then start the VM again.
  3. Log in with the username and password you created.
  4. Confirm that you can see and interact with the Parrot desktop environment.
4 Understanding Hyper-V Networking

‚è±Ô∏è ~10 minutes

Hyper-V uses virtual switches to connect virtual machines to networks. In this part, you will learn how these switches work and how they affect connectivity to the internet and to Hack The Box resources.

üåê Default and External Virtual Switches (Recommended)

Hyper-V can create virtual switches that bridge traffic between VMs, the host, and the physical network. For most Hack The Box scenarios, using the Default Switch or an External switch is appropriate because they allow internet access.

Why this matters:

  • Provides Parrot with outbound internet connectivity.
  • Allows the VM to reach HTB‚Äôs VPN endpoints.
  • Keeps configuration relatively simple for beginning labs.

Analogy: The virtual switch acts like a managed power strip for network traffic: you plug multiple VMs into one logical ‚Äúoutlet‚Äù that leads to the wider network.

üåâ External vs Internal vs Private

Hyper-V supports several types of virtual switches, each with a different visibility level.

External Switch:

  • Bridges VMs to the physical network adapter.
  • VMs can access the internet and be visible on the local LAN.
  • Suitable for HTB connectivity in many environments.

Internal Switch:

  • Allows communication between the host and its VMs only.
  • No direct internet access without additional routing.
  • Useful for host‚ÄìVM labs and internal simulations.

Private Switch:

  • Allows communication only between VMs on the same host.
  • No connection to the host or the wider network.
  • Useful for isolated multi-VM scenarios or malware labs.

üöß Considerations for Hack The Box

Hack The Box requires your Parrot VM to reach HTB‚Äôs VPN infrastructure across the internet. Therefore, the VM must have working outbound connectivity via an appropriate virtual switch.

  • For introductory labs, using the Default Switch or an External switch is recommended.
  • Internal and Private switches are not sufficient by themselves for HTB because they do not provide outbound internet access.

üîë Key Takeaways

Each virtual switch type controls what your VM can reach:

  • üåê Default/External ‚Üí Internet access and, optionally, LAN visibility.
  • üè† Internal ‚Üí Host and other VMs only.
  • üîí Private ‚Üí Only VMs on the same host.

Selecting the correct virtual switch is essential for balancing functionality, isolation, and risk in cybersecurity labs.

5 Connecting Parrot to Hack The Box

‚è±Ô∏è ~15‚Äì20 minutes

In this part, you will connect your Parrot VM to Hack The Box (HTB) using a VPN configuration file. This creates a secure tunnel from your VM to the HTB network, allowing you to reach assigned lab machines for practice and assessment.

Step 1 üõ†Ô∏è: Confirm Internet Access from Parrot

Before establishing the VPN, verify that the VM can reach the internet.

  1. Start your Parrot VM in Hyper-V and log in.
  2. Confirm in Hyper-V Manager that the VM is attached to the intended virtual switch (Default or External).
  3. Inside Parrot, open a terminal.
  4. Run: 
    ping -c 4 8.8.8.8
  5. You should see replies from 8.8.8.8. Press Ctrl+C to stop.
  6. Optional: Open the web browser (for example, Firefox) and navigate to https://www.hackthebox.com to confirm HTTP/HTTPS access.

Step 2 üõ†Ô∏è: Update Package Lists

Ensure Parrot‚Äôs package database is current.

  1. In the Parrot terminal, run: 
    sudo apt update
  2. Enter your password if prompted.
  3. Wait for the command to complete and review for any errors.

Step 3 üõ†Ô∏è: Install OpenVPN (If Needed)

Install the VPN client required for many HTB connection packs.

  1. In the same terminal, run: 
    sudo apt install openvpn -y
  2. Allow the installation to complete.
  3. When the shell prompt returns without errors, OpenVPN is available.

Step 4 üõ†Ô∏è: Download Your Hack The Box VPN Configuration

Obtain the personalized configuration that links your VM to your HTB account.

  1. From within Parrot, open a web browser.
  2. Navigate to https://www.hackthebox.com and log in.
  3. Locate the Access or Labs section associated with your subscription or lab access.
  4. Download the appropriate VPN .ovpn file for your region or assignment.
  5. Verify that the file appears in your Downloads directory in Parrot.

Step 5 üõ†Ô∏è: Navigate to the VPN File in the Terminal

Move into the directory where the HTB VPN file is stored.

  1. In the Parrot terminal, run: 
    cd ~/Downloads
  2. List the directory contents: 
    ls
  3. Identify the file ending in .ovpn, such as lab-region-username.ovpn.

Step 6 üõ†Ô∏è: Connect to Hack The Box Using OpenVPN

Establish the VPN tunnel from Parrot to HTB.

  1. Remain in the directory that contains your .ovpn file.
  2. Run the following command, substituting your actual file name: 
    sudo openvpn your_htb_config.ovpn
  3. Enter your password if prompted.
  4. Observe the terminal output. Once you see Initialization Sequence Completed, the VPN is active.
  5. Do not close this terminal window while working in Hack The Box. Closing it will terminate the VPN session.

Step 7 üõ†Ô∏è: Verify the VPN Connection

Confirm that the Parrot VM is successfully attached to the HTB network.

  1. Open a new terminal window in Parrot, leaving the OpenVPN terminal running.
  2. Run: 
    ip a
  3. Look for an interface named tun0 (or similar). This represents the VPN tunnel.
  4. In your browser, start an assigned Hack The Box machine and note its IP address.
  5. From the new terminal, test connectivity: 
    ping -c 4 <HTB_machine_IP>
  6. If you receive replies, your Parrot VM is successfully connected to the HTB environment.

Step 8 üõ†Ô∏è: Disconnect When Finished

Close the VPN tunnel in a controlled manner.

  1. Return to the terminal window where OpenVPN is running.
  2. Press Ctrl+C to stop the OpenVPN process.
  3. Confirm in the output that the VPN session is terminating.
  4. Run ip a again if you wish to verify that the tun0 interface has been removed.
