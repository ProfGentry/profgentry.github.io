Introduction

Virtual machines provide a safe, isolated environment for cybersecurity experimentation. In this lab, you will install VMware Workstation Player, deploy a Kali Linux virtual machine, configure its networking, and connect it to TryHackMe.

By the end of this activity, you will have a reusable lab environment that supports ongoing practice with tools, techniques, and procedures used in modern cybersecurity.

Learning Outcomes

  • Explain virtualization and its value in cybersecurity practice.
  • Install VMware Workstation and set up a Kali Linux virtual machine.
  • Differentiate between NAT, Bridged, and Host-Only network modes.
  • Connect Kali to TryHackMe using OpenVPN and verify connectivity.
2 VMware & Virtualization Concepts

What Is Virtualization?

Virtualization creates a simulated computing environment where multiple operating systems can run simultaneously on the same hardware without interfering with one another.

How VMware Fits In

VMware provides the “hypervisor” layer that emulates hardware resources (CPU, memory, storage, and network interfaces) for each guest virtual machine. The guest OS behaves as though it is running on its own dedicated hardware.

Why Cybersecurity Uses Virtual Machines

  • Isolate tools and malware from the host operating system.
  • Test configurations and scripts without risking production systems.
  • Practice offensive and defensive techniques in controlled environments.
  • Revert to known-good snapshots when experiments go sideways.

Why Use Kali Linux?

Kali provides a full suite of penetration testing tools and is widely adopted in security training. It is the recommended OS for TryHackMe exercises and many academic cybersecurity labs.

3 Install VMware Workstation Player

In this part of the lab you will download and install VMware Workstation Player on your Windows host. If VMware is already installed and working, quickly review these steps and move to the next tab.

Download VMware

Visit https://www.vmware.com and download VMware Workstation Player for Windows (Desktop Hypervisor section).

Install VMware

Run the installer with administrative privileges and accept the default configuration settings unless your institution provides specific guidance. When installation completes, launch VMware Workstation Player to confirm it opens without errors.

4 Create & Install Kali Linux VM

This section walks you through creating and configuring your Kali Linux virtual machine.

Download Kali ISO

Access the installer ISO at: Kali Official Downloads . Download the 64-bit installer image and save it somewhere easy to find.

Create Virtual Machine in VMware

  • Select Create a New Virtual Machine.
  • Choose Installer disc image file (ISO) and select the Kali ISO.
  • For the guest OS, choose Linux > Debian 64-bit.
  • Name the VM Kali for TryHackMe and select a storage location.
  • Allocate at least 4 GB RAM and 2 CPU cores, if available.
  • Set disk size to at least 40 GB, stored as a single file.

Complete Kali Installation

Power on the VM and choose Graphical install. Follow the prompts to set language, keyboard, hostname, and user credentials. Use guided partitioning and allow the installer to configure GRUB. When installation completes, reboot and log in to the Kali desktop.

5 Networking Modes

NAT (Recommended)

NAT shares the host’s IP address and provides internet access while keeping the VM on a private virtual network. For this lab and for TryHackMe, NAT is the default and recommended mode.

  • Internet access via the host’s connection.
  • Reasonable isolation from the local network.
  • Minimal configuration required.

Bridged

Bridged mode places the VM directly on the same network as the host, giving it its own IP address on the physical LAN.

  • Useful when the VM must appear as a separate device on the network.
  • More exposed to other devices on the LAN.
  • Requires suitable network permissions and configuration.

Host-Only

Host-only networking isolates the VM from the external network. The VM can communicate with the host and other host-only VMs, but not the internet.

  • Appropriate for offline or malware analysis labs.
  • Not suitable when the VM needs to reach TryHackMe.

High-Level Networking View

Host OS → VMware Network Adapter (NAT) → Kali VM → Internet / TryHackMe

In NAT mode, the Kali VM reaches the internet and TryHackMe via the host’s connection, while remaining logically separated from other devices on the local network.

6 Connecting to TryHackMe

This section covers installing OpenVPN, downloading your TryHackMe key, and verifying connectivity.

Install OpenVPN

Update package information and install OpenVPN in Kali.

sudo apt update
sudo apt install openvpn -y

Download Your TryHackMe VPN File

In the Kali browser, sign in to TryHackMe, navigate to the VPN/Access page, and download your personal .ovpn file to the Downloads directory.

Connect to TryHackMe

From the directory containing your VPN file, start OpenVPN:

cd ~/Downloads
sudo openvpn yourfile.ovpn

When you see Initialization Sequence Completed, the VPN tunnel is active. Leave this terminal window open while you are working on TryHackMe machines.
