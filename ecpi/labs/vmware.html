1 Understanding VMWare

üß† Understanding VMWare

The "What" Before the "How"

üí° What is Virtualization?

Virtualization is the creation of a simulated (virtual) computer environment (machine) that runs inside your physical computer. It allows you to run multiple different operating systems at the same time without them interfering with your main system.

üéØ Why Cybersecurity Uses Virtual Machines

Security professionals rely on VMs to:

  • Safely run hacking tools
  • Isolate malware
  • Test configurations
  • Recreate networks and targets

üîÑ How VMware Works

VMware simulates hardware components such as:

  • CPU
  • RAM
  • Storage
  • Network Adapters

This lets your virtual machine behave just like a real computer.

üéì Types of Virtual Machines

  • ‚ú® Full OS machines (e.g. Kali Linux)
  • üß† Specialized appliances (security-focused distros)
  • üîÑ Lightweight testing environments

üéì Why we use Kali Linux for TryHackMe

Kali includes tools such as Nmap, Burp Suite, Hydra, Gobuster, and Metasploit, making it ideal for penetration testing labs.

3 Setting Up a Kali Linux VM

‚è±Ô∏è 35-45 minutes

In this part, you will create a brand new virtual machine that will run Kali Linux. Follow each step carefully and read what each step is doing.

3.1 -üì• Download the ISO

Get the "installer disc" (iso) for Kali Linux. This is similar to downloading a Windows installer.

  1. Open a web browser in your host (Windows) system
  2. Go to https://www.kali.org/get-kali/#kali-imstaller-images
  3. Download the 64-bit "Installer" ISO image
  4. Save it somewhere easy to find (e.g. Downloads or a folder named ISOs)
3.2 -üèóÔ∏è Create a New VM in VMware

In this step, you are creating the "computer within a computer"

  1. Open VMware Workstation
  2. Click "Create a New Virtual Machine"
  3. Select "Intaller disc image file (ISO)" and browse to the Kali ISO you downloaded in the previous step.
  4. Click Next
3.3 - Choose the Guest Operating System Type

Optimize settings for the OS you are installing

  1. For "Guest Operating system," choose **Linux**
  2. For "Version" choose **Debian 10.11 64-bit** (Kali is based on Debian)
  3. Click Next
3.4 - Name the Virtual Machine

Give your VM a clear name and choose where the files will live.

  1. Enter a name like **Kali for TryHackMe**
  2. Choose a location on your disk where you have enough space (e.g., a folder like C:\VMs\Kali-THM)
  3. Click Next
3.5 - Specify Disk Capacity

Decide how big your virtual hard drive will be.

  1. Set the maximum disk size to at least 40GB (Don't worry to much, you can actually change this later if needed)
  2. Choose "Store virtual disk as a single file" (simpler for most users)
  3. Click Next
3.6 - Customize Hardware

Allocate part of your host computer's resources to the Virtual Machine

  1. Click "Customize Hardware..." before finishing
  2. Memory (RAM): Set to 4096 MM (4GB) minimum if your host has at least 8 GB.
  3. Processors: Set to 2 cores if available
  4. Network Adapter: Leave as NAT (this lets the VM share your host's internet connection safely.
  5. Click Close, then Finish
3.7 - Power On the VM and Start the Installer üî•‚û°Ô∏èüêß

Boot from the virtual "DVD" to install the operating system.

  1. Make sure your Kali VM is selected in VMware
  2. Click "Play virtual machine."
  3. When the Kali boot menu appears, choose Graphical install using the keyboard and press Enter
3.8 - Run Through the Kali Installer

Install the Kali operating system onto the virtual hard drive.

  1. Select your language, location, and keyboard layout.
  2. Enter a hostname (e.g., kali)
  3. Create a user account and password. Remember this password.
  4. For partitioning, select "Guided - use entire disk"
  5. Accept the defaults when asked where to install the bootloader (GRUB)
  6. Wait while the system copies files and installs the desktop environment
3.9 - Reboot into Kali Desktop

Finish the installation and start Kali for real.

  1. When installation is done, select "Continue" to reboot.
  2. Log in with the username and password you created.
  3. LYou should see the Kali Linux desktop
4 Understanding VM Networking Modes

‚è±Ô∏è 10 minutes

üé¨ NAT (Recommended)

In this part, you will understand how your virtual machine communicates with the outside world and with your host system. These modes determine whether your Kali VM can access the internet, talk to your local network, or stay isolated for safe testing.

üé¨ NAT (Recommended)

NAT (Network Address Translation) allows your VM to share your host computer‚Äôs internet connection. The VM gets a private internal IP address, and VMware acts like a router translating traffic.

Why this matters:

  • Safest option for beginners
  • Perfect for TryHackMe VPN access
  • Your VM stays hidden from your real network, reducing risk

Analyogy:

Think of NAT like your VM using your host‚Äôs Wi-Fi password without actually being given the password.

üé¨ Bridged

Bridged mode places your VM directly on your physical network as if it's its own separate machine with its own IP address.

Why this matters:

  • Useful when simulating real networks
  • Helpful for labs requiring your VM to appear as a separate device
  • Less isolated, so slightly riskier for beginners

strong>Analogy:

Your VM becomes a ‚Äúreal computer roommate‚Äù sharing your network. Visible, active, and discoverable.

üé¨ Host Only

This mode isolates your VM from the internet entirely. The VM can only communicate with your host computer.

Why this matters:

  • Great for malware analysis (nothing escapes)
  • Secure for offline testing
  • Not suitable for TryHackMe since internet is required

Analogy:

Host-Only mode is like airplane mode for your VM‚Ä¶ but it can still talk to you sitting next to it.
