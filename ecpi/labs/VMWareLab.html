Lab Overview

Virtual machines allow you to run additional operating systems in a controlled, isolated environment on a single physical computer. This capability is foundational in cybersecurity education and practice, where experimentation and safe testing are critical.

In this laboratory exercise, you will install VMware Workstation Player, create and configure a Kali Linux virtual machine, adjust its networking mode, and connect it to TryHackMe. At the conclusion of the lab, you will have a fully functional environment suitable for hands-on cybersecurity exercises.

1 Introduction to VMware and Virtualization

Approximate time: 10 minutes

Conceptual Background

Before working with the tools, it is important to understand what virtualization is and how VMware fits into the broader computing environment. This section provides a concise conceptual overview.

What Is Virtualization?

Virtualization is the process of creating a simulated (virtual) computing environment that runs within a physical host system. It allows multiple isolated operating systems to run concurrently on the same hardware without interfering with each other.

Why Cybersecurity Uses Virtual Machines

Security professionals and students rely on virtual machines to:

  • Run security tools without altering the host operating system.
  • Isolate potentially malicious software for analysis.
  • Test system and network configurations safely.
  • Simulate networks and target systems for training and assessment.

How VMware Works

VMware provides a virtual hardware layer that emulates key components of a physical computer, including:

  • CPU
  • Memory (RAM)
  • Storage (virtual disks)
  • Network interfaces

The guest operating system (e.g., Kali Linux) interacts with this virtualized hardware as if it were running on a dedicated physical machine.

Types of Virtual Machines

  • Full operating system environments (e.g., Kali Linux, Windows, Ubuntu).
  • Pre-configured virtual appliances for specific security or network roles.
  • Lightweight test environments for isolated experimentation.

Why Use Kali Linux in This Lab?

Kali Linux is a Debian-based distribution designed for penetration testing and security analysis. It includes tools such as Nmap, Burp Suite, Hydra, Gobuster, and Metasploit, making it well-suited for TryHackMe and similar platforms.

3 Creating and Installing a Kali Linux Virtual Machine

Approximate time: 35–45 minutes

In this part, you will create a new virtual machine that runs Kali Linux. Read each step carefully; the descriptions explain the purpose of each configuration choice.

3.1 Download the Kali Linux Installer ISO

The installer ISO file is the image from which the operating system will be installed.

  1. Open a web browser on the host (Windows) system.
  2. Go to the official Kali download page: https://www.kali.org/get-kali/#kali-installer-images.
  3. Download the 64-bit “Installer” ISO image.
  4. Save the file in an easy-to-find location (e.g., Downloads or a dedicated ISOs folder).
3.2 Create a New Virtual Machine in VMware

In this step, you define the basic virtual machine that will host Kali Linux.

  1. Open VMware Workstation Player.
  2. Select Create a New Virtual Machine.
  3. Choose Installer disc image file (ISO) and browse to the Kali ISO downloaded in Step 3.1.
  4. Select the ISO and click Next.
3.3 Select Guest Operating System Type

Choosing the correct guest OS type helps VMware apply appropriate default settings.

  1. For Guest operating system, select Linux.
  2. For Version, choose Debian 10.x 64-bit (Kali is Debian-based).
  3. Click Next.
3.4 Name the Virtual Machine

Provide a clear name and choose a storage location for the VM files.

  1. Enter a VM name such as Kali for TryHackMe.
  2. Select a folder on disk with sufficient free space (e.g., C:\VMs\Kali-THM).
  3. Click Next.
3.5 Specify Disk Capacity

Determine the size and layout of the virtual hard disk.

  1. Set the maximum disk size to at least 40 GB.
  2. Choose Store virtual disk as a single file (this is simpler for most users).
  3. Click Next.
3.6 Customize Virtual Hardware

Allocate host resources to the VM to support Kali effectively.

  1. Before finishing, click Customize Hardware….
  2. Set Memory (RAM) to at least 4096 MB (4 GB), assuming your host has at least 8 GB RAM.
  3. Set Processors to 2 cores if available.
  4. Ensure the Network Adapter is set to NAT (recommended for this lab).
  5. Click Close, then click Finish to complete VM creation.
3.7 Power On the VM and Start the Installer

Boot the VM using the ISO and launch the Kali installer.

  1. Select the newly created Kali VM in VMware.
  2. Click Play virtual machine.
  3. When the Kali boot menu appears, select Graphical install and press Enter.
3.8 Complete the Kali Installation

Install Kali Linux onto the virtual disk.

  1. Choose language, location, and keyboard layout as appropriate.
  2. Specify a hostname (e.g., kali).
  3. Create a user account and password; record these credentials securely.
  4. For partitioning, select Guided – use entire disk.
  5. Accept the defaults for installing the bootloader (GRUB).
  6. Allow the system to copy files and complete installation of the desktop environment.
3.9 Reboot into the Kali Desktop

Finalize installation and verify that the desktop loads correctly.

  1. When prompted, select Continue to reboot.
  2. Log in using the credentials you created.
  3. Confirm that the Kali Linux desktop appears and is responsive.
4 Virtual Machine Networking Modes

Approximate time: 10 minutes

Overview of Networking Modes

Virtual machine networking determines how the guest operating system communicates with the internet, the local network, and the host system. Understanding these modes helps you select an appropriate configuration for different cybersecurity scenarios.

NAT (Network Address Translation) — Recommended for This Lab

NAT allows the VM to share the host’s internet connection while remaining on an internal virtual network. VMware performs address translation, so the VM uses a private IP address behind the host.

Characteristics:

  • Suitable for most introductory labs and online platforms such as TryHackMe.
  • Provides internet access while reducing direct exposure to the local network.
  • Typically requires no special configuration on the physical network.

Bridged Networking

Bridged mode places the VM directly on the same physical network as the host, as if it were another physical machine with its own IP address.

Characteristics:

  • Useful when the VM must appear as an independent device on the local network.
  • Facilitates certain advanced network simulations and services.
  • Less isolated; use with care in untrusted environments.

Host-Only Networking

Host-only mode isolates the VM from the external network. The VM can communicate only with the host and other VMs configured for the same host-only network.

Characteristics:

  • Appropriate for offline testing and certain malware analysis scenarios.
  • No direct internet connectivity from the VM.
  • Not suitable for labs requiring external connectivity, such as TryHackMe VPN access.

Key Takeaways

  • NAT — Internet access via host; safer default for most exercises.
  • Bridged — Full participation on the local network; use when explicitly required.
  • Host-Only — Isolated environment; appropriate when internet access is not desired.

For this lab, NAT is the recommended mode to support TryHackMe connectivity while maintaining a reasonable degree of isolation.

5 Connecting Kali to TryHackMe

Approximate time: 15–20 minutes

Objective

In this part of the lab, you will verify network connectivity from the Kali VM, install OpenVPN, obtain your TryHackMe VPN configuration file, and establish a secure VPN tunnel to the TryHackMe network.

Step 1: Verify Basic Internet Connectivity

Confirm that your Kali VM can reach the internet before configuring the VPN.

  1. Start your Kali VM in VMware.
  2. In VMware, verify that the VM’s Network Adapter is set to NAT.
  3. Inside Kali, open a terminal.
  4. Run the following command:
ping -c 4 8.8.8.8
  1. You should see ICMP replies from 8.8.8.8. Press Ctrl+C to stop the ping.
  2. Optionally, open Firefox in Kali and browse to https://tryhackme.com to confirm web access.

Step 2: Update Kali Package List

Refresh the package index so that the latest software versions are available.

  1. In the Kali terminal, run:
sudo apt update
  1. Enter your password if prompted.
  2. Allow the command to complete; this step does not install software yet.

Step 3: Install OpenVPN

Install the VPN client required for TryHackMe connectivity.

  1. In the same terminal, run:
sudo apt install openvpn -y
  1. Wait while Kali downloads and installs OpenVPN and any dependencies.
  2. When the prompt returns without errors, OpenVPN is installed.

Step 4: Download Your TryHackMe VPN Configuration File

Obtain a personalized .ovpn file linked to your TryHackMe account.

  1. In Kali, open Firefox.
  2. Navigate to https://tryhackme.com and sign in.
  3. Open the Access or VPN section of the site.
  4. Select the OpenVPN option if prompted.
  5. Download your .ovpn file.
  6. Confirm that the file is saved (typically in the Downloads directory in Kali).

Step 5: Navigate to the VPN File in the Terminal

Move to the directory containing the VPN configuration file.

  1. In the Kali terminal, change to the Downloads folder:
cd ~/Downloads
  1. List the files to confirm the presence of the .ovpn file:
ls
  1. Identify the VPN file, which typically has a name similar to thm-region-username.ovpn.

Step 6: Connect to TryHackMe Using OpenVPN

Establish a secure VPN tunnel between Kali and the TryHackMe network.

  1. Ensure you are still in the directory where the .ovpn file is located.
  2. Run the following command, replacing the filename with your actual configuration file:
sudo openvpn yourfile.ovpn
  1. Enter your password if prompted.
  2. Observe the terminal output as OpenVPN starts the connection process.
  3. When you see the line Initialization Sequence Completed, the VPN connection is active.
  4. Important: Do not close this terminal window while you wish to remain connected.

Step 7: Verify the VPN Connection

Confirm that the VM is successfully on the TryHackMe network.

  1. Open a new terminal window or tab in Kali (leave the OpenVPN terminal running).
  2. Display the network interfaces:
ip a
  1. Verify that an interface named tun0 is present; this is the VPN tunnel.
  2. In your browser, start a TryHackMe machine that provides an IP address.
  3. From the new terminal, test connectivity to the machine’s IP address (replace with the actual IP):
ping -c 4 <THM_machine_IP>
  1. If you receive replies, your Kali VM is successfully connected to the TryHackMe network.

Step 8: Disconnect at the End of the Session

Terminate the VPN connection when you are finished working.

  1. Return to the terminal window where OpenVPN is running.
  2. Press Ctrl+C to stop the OpenVPN process.
  3. Confirm that messages indicate the connection has been terminated.
  4. Verify that the tun0 interface no longer appears in ip a.

Lab Summary and Reflection

What You Accomplished

  • Explained the concept of virtualization and its relevance to cybersecurity practice.
  • Installed VMware Workstation and deployed a Kali Linux virtual machine.
  • Configured virtual networking modes and selected an appropriate mode for TryHackMe.

Suggested Next Steps

  • Install additional productivity tools in Kali (e.g., an enhanced terminal emulator).
  • Customize the Kali environment to suit your workflow (panels, themes, preferred tools).
  • Experiment with shared folders or other host–guest integration mechanisms.
  • Complete introductory TryHackMe rooms to apply what you have configured.

Important Considerations

  • Isolation is relative: Virtual machines improve safety but do not eliminate risk; follow institutional guidelines.
  • Snapshots: Use snapshots strategically to preserve known-good states before major changes.
  • Resource allocation: Be mindful of how CPU, memory, and disk assignments affect both host and guest performance.
  • Network configuration: Choose networking modes intentionally based on the lab’s goals and risk profile.
